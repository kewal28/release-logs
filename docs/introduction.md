# Introduction

Release Log is a developer-focused **changelog / release notes** platform. You can **self-host** it or run a hosted (SaaS) variant. Visitors see a clean, card-based feed; your team manages entries from an **admin dashboard**.

## Who it is for

- Startups and product teams publishing release notes
- Agencies managing multiple products (**User → Project → Changelog**)
- Developers who want a **small embeddable widget** (script snippet) on their marketing site or app

## Key benefits

- **Multi-project**: One account can own several projects (e.g. product A vs B), each with its own **public embed key** so the correct feed loads on each site.
- **Feedback**: Comments and votes on published entries (configurable moderation).
- **Docs-first**: This `/docs` folder syncs to **GitBook** for public, versioned documentation.

The **canonical live API description** for your deployment is always `{BASE_URL}/api-docs` (Swagger UI).
